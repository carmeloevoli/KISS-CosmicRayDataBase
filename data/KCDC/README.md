# KCDC Spectra Files

This folder contains local text exports derived manually from the KASCADE Cosmic Ray Data Centre (KCDC) spectra interface and used by KISS as an auxiliary source for KASCADE and KASCADE-Grande spectra.

## Provenance

- Main project page: `https://kcdc.iap.kit.edu/`
- Spectra interface: `https://kcdc.iap.kit.edu/accounts/login/?next=/spectra/`
- Access note: the spectra page requires authentication, so these files were extracted manually rather than generated by an automated script.
- Last checked: April 21, 2026. 

## Why These Files Exist

KISS uses CRDB as the primary source when possible. For some KASCADE and KASCADE-Grande spectra, the material used in this repository was taken from the KCDC spectra portal instead. This folder keeps those hand-derived tables together with enough context to trace where they came from.

## File Naming

The filenames follow the pattern

`<experiment>_<description>_<quantity>_totalEnergy.txt`

Examples:

- `KASCADE_2005_QGSJET-01_H_totalEnergy.txt`
- `KASCADE_2011_SIBYLL-2.1_allParticle_totalEnergy.txt`
- `KASCADE-Grande_QGSJET-II-04_light_totalEnergy.txt`

Interpretation:

- `<experiment>` is `KASCADE` or `KASCADE-Grande`.
- `<description>` usually identifies the publication or release year and the hadronic interaction model shown in KCDC.
- `<quantity>` is the spectrum stored in the file, for example `H`, `He`, `light`, or `allParticle`.
- `totalEnergy` reflects how these data are interpreted inside KISS.

The exact physics meaning of labels such as `light` follows the KCDC spectra page and the underlying KASCADE/KASCADE-Grande publication. If you reuse a file outside KISS, it is worth checking the corresponding paper or KCDC plot description.

## File Format

Each file is an ASCII table with six metadata lines followed by semicolon-separated numeric data. A typical header is:

```text
#InGeV: 1
#InLog10: 0
#Scaling: 0.0
#Label: ...
#Columns: E, Flux, UncertLow, UncertHigh
#Separator: ";"
```

The data rows have four columns:

1. `E`
2. `Flux`
3. `UncertLow`
4. `UncertHigh`

Notes:

- `#InGeV: 1` indicates that the energy axis is stored in GeV.
- `#InLog10: 0` indicates that the tabulated values are linear, not `log10`.
- `#Scaling: 0.0` indicates no extra energy scaling in the stored flux values.
- The separator is `;`.

## How KISS Reads These Files

The reader is implemented in `src/KCDC.cpp`.

Current assumptions in the parser:

- the first six lines are skipped as header;
- each remaining line contains exactly four semicolon-separated values;
- the columns are interpreted as `E`, `Flux`, `UncertLow`, `UncertHigh`;
- the x-axis is treated as a total-energy point list for the corresponding dataset.

If the local file format changes, `src/KCDC.cpp` will likely need to be updated as well.

## Maintenance Notes

- If a file is replaced or redigitized, keep the provenance trace here.
- If possible, record the exact KCDC plot or publication used to derive the table.
- When adding new files, try to preserve the same header structure so they remain compatible with the current parser.
